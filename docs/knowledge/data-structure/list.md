리스트

리스트는 원소를 순서대로 저장하는 자료구조를 말한다(선형을 이루고 있다.)
배열기반 리스트는 물리적논리적 선형을 이루고 있고
포인터기반 리스트는 논리적 선형을 이루고 있다

리스트는 중간에 빈공간이 있으면 안된다.
따라서 중간 삭제가 발생하는경우 빈공간을 메꾸기위한 적절한 조치가 필요하다.

리스트의 종류
-배열기반 리스트
배열기반 리스트는 배열을 사용하여 리스트를 구현한것이다.
배열의 특성상 최대 개수가 고정되어 있고
중간에 삽입 삭제시 메모리의 잦은 이동이 발생한다. 따라서 빈번히 삽입 삭제가 이루어지는 상황에서는 추천되지 않는다.
배열은 메모리상 순차적으로 저장되므로 순회시에 높은 성능을 낼수있다.
또한 물리적 논리적 배치가 같기때문에 인덱스를통한 상수접근이 가능하다.


-포인터기반 리스트(링크드리스트)
최대 저장개수가 정해지지 않기에 자원이 허락하는한 추가될수있다.
중간 삽입 삭제시에 포인터 정보만 바꾸어주면 되므로 빈번히 삽입삭제가 발생한다면 포인터기반 리스트가 좋다
원소가 논리적으로만 선형을 이루고 있기때문에 인덱스접근이 불가능하고 순회하면서 접근해야한다.

포인터기반 리스트의 종류로는
단순 링크드 리스트, 원형 링크드 리스트, 이중 링크드 리스트
가 있다.

링크드 리스트의 구현에 있어서
헤더를 값노드로할지 포인터링크로 할지에따라 구현난이도가 달라진다.
값노드로 하는경우가 분기가 적어 쉬운반면 메모리를 조금더 쓴다.
포인터링크로하는경우가 분기가 많아 어려운반면 비교적 메모리를 덜 쓴다.

단순 링크드 리스트는 단순히 단방향으로 연결정보가 나열되어 있다.
만약 현재 노드 이전 노드에 접근하려면 첫번재 노드부터 순회를 돌아야한다.

원형 링크드 리스트는 마지막 노드의 연결정보가 NULL이 아닌 첫번째 노드를 바라보고 있는 순환 구조이다.
만약 현재 노드 이전 노드에 접근하려면 계속 링크를 순회하면서
pPreNode.pLink == pNode 조건을 만족하는 노드를 찾아야한다.
만약 헤더포인터기반 으로 구현한 원형링크드리스트에서는 
원소 추가시 분기가 3개로 나뉜다
case 1-1:첫번째 노드 추가 && 공백 리스트인경우
case 1-2:첫번째 노드 추가 && 공백 리스트가 아닌 경우
case 2: 첫번째 노드 추가 아닌경우
원소 삭제시도 분기가 3개로 나뉜다
case 1-1: 첫번째 노드 제거 && 마지막 원소인경우
case 1-2: 첫번째 노드 제거 && 마지막 원소가 아닌 경우
case 2: 첫번째 노드 제거가 아닌경우

이중 링크드 리스트는 연결정보가 양방향이기때문에 현재 노드 이전 노드에 접근할때 그냥 왼쪽 링크정보를 타면된다.

책에서 배웠던 링크드 리스트의 추가기능으로
링크드 리스트2개를 연결하는기능과, 링크드 리스트 역순화가 있다.

링크르 리스트 응용의 예로
다항식 계산이 있었다.
